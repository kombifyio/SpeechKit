# Code of Conduct

## Our Standard

We expect contributors and maintainers to keep discussion direct, respectful, and technically grounded. SpeechKit is built for practical collaboration. That means:

- critique code and assumptions, not people
- explain tradeoffs clearly
- avoid harassment, intimidation, or hostile pile-ons
- respect different experience levels and backgrounds
- keep security-sensitive information private

## Unacceptable Behavior

The following are not acceptable in project spaces:

- personal attacks, insults, or demeaning language
- discriminatory language or conduct
- deliberate intimidation or harassment
- publishing private or security-sensitive information without permission
- repeated bad-faith disruption of issues, reviews, or release work

## Maintainer Responsibilities

Maintainers may edit, hide, lock, or remove content that violates this policy. They may also restrict participation when necessary to protect contributors or the release process.

## Scope

This code of conduct applies to:

- issues
- pull requests
- discussions
- release coordination
- other project communication channels operated for SpeechKit

## Reporting

For behavior issues, contact the maintainers through the repository owners or the private maintainer channel used for this repo. For vulnerabilities, use the process in [`SECURITY.md`](./SECURITY.md).
